# Исправление ограничения размера данных

## Описание проблемы

Пользователь запустил команду:
```bash
mpirun -n 1 ./wbc1_original_parallel 1 256 0 32 1 100000
```

**Ожидаемый результат:** Программа должна использовать 100000 КБ (100 МБ) данных для статистического анализа.

**Фактический результат:** Программа использовала только 1000 КБ (1 МБ) данных.

```
Data size / Размер данных: 1000 KB  ← НЕПРАВИЛЬНО (должно быть 100000 KB)
```

## Первопричина

В файле `wbc1_original_parallel.c` были найдены два ограничения размера данных:

1. **Строка 645 (task 0 - шифрование текста):**
   ```c
   if (data_kb > 10000) data_kb = 10000;  /* Max 10MB */
   ```

2. **Строка 738 (task 1 - статистический анализ):**
   ```c
   if (data_kb > 1000) data_kb = 1000;  /* Max 1MB - СЛИШКОМ МАЛО! */
   ```

Пользователь запускал task 1 (статистический анализ), который был ограничен 1000 КБ.

## Решение

Увеличены оба ограничения до 100000 КБ (100 МБ) для обеспечения согласованности и поддержки больших датасетов.

### Изменения кода

**До:**
```c
// Task 0 (строка 645)
if (data_kb > 10000) data_kb = 10000;  /* Max 10MB */

// Task 1 (строка 738)
if (data_kb > 1000) data_kb = 1000;  /* Max 1MB */
```

**После:**
```c
// Task 0 (строка 645)
if (data_kb > 100000) data_kb = 100000;  /* Max 100MB */

// Task 1 (строка 738)
if (data_kb > 100000) data_kb = 100000;  /* Max 100MB */
```

## Проверка исправления

### Команды для тестирования

```bash
# Компиляция (если необходимо)

# Тест с 100 МБ (100000 КБ)
mpirun -n 1 ./wbc1_original_parallel 1 256 0 32 1 100000

# Тест с 50 МБ (50000 КБ)
mpirun -n 1 ./wbc1_original_parallel 1 256 0 32 1 50000

# Тест с 10 МБ (10000 КБ)
mpirun -n 1 ./wbc1_original_parallel 1 256 0 128 1 10000
```
